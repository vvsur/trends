# Backlog корпоративного портала трендов

## Легенда приоритетов

- P0: необходимо для MVP и KPI 2026.
- P1: важно для устойчивого процесса 2027.
- P2: масштабирование и прогнозный контур 2028+.

## Epic 1. Базовая структура портала

| ID | Приоритет | User story | Acceptance criteria |
| --- | --- | --- | --- |
| TR-001 | P0 | Как пользователь, я вижу главную страницу портала с ключевыми разделами | Есть навигация: техрадар, реестр, скоринг, пилоты, KPI, кейсы |
| TR-002 | P0 | Как администратор, я управляю справочниками департаментов, доменов и статусов | Можно создать/изменить/деактивировать справочник без изменения кода |
| TR-003 | P0 | Как руководитель, я вижу сводку KPI 1.3 | Отображаются квартальный скоринг, пилоты по департаментам, соблюдение дорожной карты |

## Epic 2. Корпоративный радар трендов

| ID | Приоритет | User story | Acceptance criteria |
| --- | --- | --- | --- |
| TR-010 | P0 | Как владелец тренда, я создаю карточку тренда | Карточка содержит название, описание, домен, зрелость, рекомендацию, владельца, дату пересмотра |
| TR-011 | P0 | Как пользователь, я фильтрую тренды по домену, статусу, владельцу и департаменту | Фильтры работают совместно и сохраняются в URL |
| TR-012 | P1 | Как пользователь, я вижу радарную визуализацию трендов | Тренды распределены по кольцам зрелости и квадрантам доменов |
| TR-013 | P1 | Как владелец тренда, я фиксирую источники сигналов | У тренда есть список источников, дата сигнала и краткая интерпретация |
| TR-014 | P0 | Как пользователь, я различаю технологические, биржевые, финансовые, HR, регуляторные и другие домены | Домен является обязательным полем, отображается в карточке, фильтрах и отчетах |
| TR-015 | P1 | Как руководитель, я вижу cross-domain тренды | Один тренд может быть связан с несколькими доменами и показывает влияние на бизнес, технологии, риск и HR |

## Открытые вопросы

- Какой корпоративный инструмент должен быть источником пользователей и оргструктуры?
- Нужно ли интегрироваться с Jira/Confluence/портфельной системой в MVP?
- Кто утверждает финальную модель скоринга и веса?
- Нужна ли классификация по биржевым бизнес-доменам уже в MVP?
- Какие данные можно использовать для ROI без ручного ввода?
- Какие источники официально разрешены для открытого сборщика?
- Каким способом физически передавать подписанные пакеты между открытым и закрытым контуром?
- Кто является владельцем review по каждому домену трендов?
