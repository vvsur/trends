## Регламент скоринга и ограничения

В PDF явно указано:

- "Регламент для скоринга инноваций: будет опубликован позднее".
- "Ограничения: будет опубликовано позднее".

Следствие для реализации:

- нельзя считать текущую ICE-модель финальным регламентом;
- MVP может использовать configurable draft scoring model;
- нужно предусмотреть настройку весов, версионирование и замену модели скоринга;
- финальные ограничения должны быть внесены после публикации регламента.

Покрытие:

- TR-033: настройка весов скоринга;
- TR-034/TR-035: история и ручная переоценка;
- TR-036: загрузить и применить финальный регламент скоринга и ограничения после публикации.

## Реализационные выводы

1. Не начинать MVP с пустой базы: нужен seed-импорт стратегических инициатив и первичных трендов из этого документа.
2. Не считать скоринг финальным: финальный регламент и ограничения ожидаются позднее.
3. Не реализовывать устаревшую BPMN-схему без подтверждения владельца процесса.
4. В MVP важно сохранить поля источника: департамент, квартал создания, ответственный, год, комментарий.
5. Для value/adoption части особенно важны инициативы: обучение, система поощрения, PR инновационной культуры, библиотека бизнес-кейсов.
